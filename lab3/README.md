# Лабораторная работа №3: Игра "Super Mario Bros"
**Документация системы 2D-платформера**

## 1. Общее описание
Данная программная система представляет собой 2D-платформер, вдохновленный классической игрой Super Mario Bros. Проект реализован с использованием графической библиотеки Pygame и предлагает полноценный игровой цикл: главное меню, прохождение уровня, взаимодействие с врагами и бонусами, систему подсчета очков и времени, а также таблицу рекордов.

Проект построен на основе объектно-ориентированного подхода с использованием паттерна "Состояния" для управления экранами игры (меню, игра, пауза, экран окончания игры). Дизайн уровней и настройки вынесены в JSON-файлы, что делает архитектуру гибкой и расширяемой.

## 2. Структура проекта
Файловая структура проекта организована следующим образом:

```text
Lab3/
├── main.py                 # Точка входа в программу, контроллер состояний игры
├── sources/                # Основная бизнес-логика и модели данных
│   ├── entities.py         # Игровые сущности (Игрок, Враги, Бонусы и т.д.)
│   ├── level_parser.py     # Парсер уровня, физика и логика коллизий
│   └── records.py          # Менеджер таблицы рекордов
├── sources/json/           # Конфигурационные файлы
│   ├── settings.json       # Глобальные настройки (разрешение, пути к ассетам)
│   ├── level1.json         # Карта уровня в виде двумерного массива символов
│   └── records.json        # Файл сохранения рекордов игроков
└── icons/ & music/         # Директории с графическими и звуковыми ассетами
```

## 3. Описание модулей
### 3.1. Пакет sources
#### level_parser.py
Содержит логику построения уровня и обработки физики игрового мира.

- **Block**: Наследуется от `pygame.sprite.Sprite`. Представляет собой блоки уровня (обычные или блоки со знаком вопроса, скрывающие бонусы/монеты).
- **Level**: Основной класс управления игровым процессом на конкретном уровне.
  - Парсит `level1.json`, расставляя сущности (`P` - игрок, `X` - блок, `?` - бонусный блок, `E` - враг, `F` - финиш и др.) по сетке с заданным размером (`block_size`).
  - Управляет группами спрайтов (с использованием `pygame.sprite.Group`).
  - Обрабатывает коллизии, физику падения (гравитацию) и скроллинг экрана (движение камеры).

#### records.py
Отвечает за персистентность данных рекордов (сохранение состояния между сессиями).

- **RecordsManager**: Класс, управляющий таблицей лучших игроков.
  - Сохраняет и загружает данные из `records.json`.
  - Предоставляет методы `add_record` для добавления новых достижений и `is_new_record` для проверки, вошел ли текущий счет в топ-5 лучших.
  - Автоматически сортирует рекорды по убыванию очков.

#### entities.py (на основе импортов)
Определяет иерархию игровых объектов:
- **Player**: Главный герой (имеет скорость, гравитацию, состояние "супер-силы").
- **Enemy**: Враг с простейшим ИИ (движение по прямой с отскоком от препятствий и краев платформ).
- **Bonus / BlockCoin / Goal**: Интерактивные объекты, дающие очки, время или завершающие уровень.

### 3.2. Главный модуль
#### main.py
Точка входа в приложение и глобальный менеджер игры (`Game`).

- Инициализирует Pygame, шрифты, микшер для звуков и систему дисплея.
- **Конечный автомат (State Machine)**: Управляет текущим состоянием игры через свойство `self.state` (`menu`, `playing`, `records`, `help`, `game_over`, `level_completed`, `name_input`).
- **Обработка событий**: Метод `handle_events` перехватывает нажатия клавиш для навигации по меню, паузы, полноэкранного режима (F11) и ввода имени для рекорда.
- **Отрисовка**: Метод `draw` отвечает за рендеринг нужного экрана в зависимости от состояния (вывод текста, фонов, счетчиков очков и времени).

## 4. Ключевые алгоритмы и реализация
### Физика и коллизии (AABB)
Разрешение коллизий игрока разделено на две независимые фазы (горизонтальную и вертикальную), что предотвращает "залипание" на углах платформ:
1. `horizontal_movement_collision`: Смещение по оси X. Если есть столкновение с блоком, игрок выталкивается вплотную к соответствующей стороне блока.
2. `vertical_movement_collision`: Применение гравитации и смещение по оси Y. При столкновении с блоком снизу сбрасывается вертикальная скорость (приземление). Столкновение с блоком `?` сверху инициирует его активацию (выброс монеты).

### Скроллинг камеры (World Shift)
Вместо движения камеры по координатам реализован сдвиг всего мира:
- В методе `scroll_x` отслеживается позиция игрока. Если он достигает левой (1/4 экрана) или правой (3/4 экрана) границы, скорость игрока обнуляется, а переменной `world_shift` присваивается скорость игрока (в противоположном направлении).
- Все спрайты (блоки, враги, облака) смещаются на величину `world_shift` на каждом кадре.

### Игровой цикл и управление временем
Игровой цикл реализован в методе `run` класса `Game`. Скорость обновления фиксирована настройкой `fps` (по умолчанию 60). Таймер уровня `self.time_left` уменьшается пропорционально кадрам. Если таймер доходит до нуля, вызывается смерть игрока.

## 5. Инструкция по запуску
### Требования
- Python 3.8 или выше.
- Установленная библиотека Pygame.

Установка зависимостей:
```bash
pip install pygame
```

### Запуск приложения
Находясь в корневой директории проекта, выполните команду:
```bash
python main.py
```

### Управление в игре
- **Стрелки влево/вправо** - движение.
- **Стрелка вверх / ПРОБЕЛ** - прыжок.
- **F11** - переключение полноэкранного режима.
- **ESC** - пауза (во время игры) или выход в меню.
- **R / H** - в главном меню открывают таблицу рекордов и справку соответственно.
