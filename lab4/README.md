# Лабораторная работа №4: Веб-интерфейс модели университета
**Документация клиент-серверного приложения**

## 1. Общее описание
Данная программная система представляет собой клиент-серверное веб-приложение для автоматизации процессов управления университетом. За основу взята бизнес-логика, разработанная в лабораторной работе №1. Консольный интерфейс заменен на современный браузерный интерфейс (UI) с использованием фреймворка **Flask**. 

Проект демонстрирует принцип "Разделения ответственности": бизнес-логика, модели данных и логика сохранения (база данных) полностью отделены от интерфейса взаимодействия с пользователем. Это позволяет одновременно использовать как консольную (CLI), так и веб-версию (Web) приложения, работающих с одними и теми же данными.

## 2. Структура проекта
Файловая структура проекта расширена и теперь организована следующим образом:

```text
lab4/
├── CLI/                        # Оставлено для демонстрации работы консольного интерфейса
│   ├── main.py                 # Точка входа в CLI-версию
│   └── handle_functions.py     # Консольные обработчики ввода-вывода
├── sources/                    # Ядро системы (общий код для CLI и Web)
│   ├── exceptions.py           # Пользовательские классы исключений
│   ├── models.py               # Основные сущности (студент, преподаватель и др.)
│   ├── storage.py              # Логика сохранения в JSON
│   └── university.py           # Главный класс-контейнер
├── Web/                        # Веб-интерфейс (новое в ЛР №4)
│   ├── app.py                  # Основной веб-сервер Flask (контроллер)
│   ├── static/                 # Статические файлы оформления и клиентской логики
│   │   ├── script.js           # Клиентский JavaScript
│   │   └── style.css           # Таблицы стилей CSS
│   └── templates/              # HTML-шаблоны страниц (представления)
│       ├── index.html          # Главная страница (дашборд и формы)
│       ├── student_detail.html # Страница профиля студента
│       └── teacher_detail.html # Страница профиля преподавателя
└── university.json             # Общая база данных (создаётся автоматически)
```

## 3. Описание модулей
### 3.1. Пакет Web (Новый функционал)
Слой маршрутизации и графического интерфейса, построенный на базе микрофреймворка Flask.

#### app.py
Основной файл веб-приложения, выполняющий роль "Контроллера".
- Инициализирует Flask-приложение и подключает общую логику из `sources/storage.py`.
- Обрабатывает HTTP-запросы GET (для отображения страниц) и POST (для обработки данных из форм).
- Использует механизм `flash` для передачи всплывающих уведомлений (успех/ошибка) на страницу после выполнения действий.
- Содержит маршруты для всех действий (`/add_student`, `/conduct_exam`, `/library_action` и т.д.), которые принимают параметры из HTML-форм (`request.form.get`), вызывают методы модели `University` и сохраняют изменения.
- Реализует маршрутизацию с параметрами (например, `/student/<student_id>`) для отображения детальной информации о конкретных объектах, с обработкой ошибки `404 Not Found`.

#### templates/
Папка содержит шаблоны страниц, использующие движок Jinja2 для динамического рендеринга данных:
- **index.html**: Главная страница. Разделена на две основные части: отображение текущего состояния базы данных (в виде сетки Grid) и набор форм для каждого из доступных действий (добавление, удаление, экзамены и т.д.).
- **student_detail.html / teacher_detail.html**: Шаблоны детального представления. Выводят полную информацию о выбранном человеке (зачетная книжка, стипендия, список взятых книг, читаемые предметы и т.д.).

#### static/
- **style.css**: Описывает внешний вид приложения (шрифты, цвета, отступы). Для создания адаптивного дизайна используется технология CSS Grid (`.stats-grid`, `.grid-forms`).
- **script.js**: Содержит функцию `toggleLibFields()`, которая динамически скрывает и показывает поля в форме "Библиотека" в зависимости от выбранного действия (взять/вернуть книгу или добавить в ассортимент), синхронизируя интерфейс при загрузке страницы (`DOMContentLoaded`).

### 3.2. Пакет sources (Общий код)
Пакет полностью переиспользован из ЛР №1 без изменений, что доказывает правильность заложенной архитектуры.

- **models.py**: Сущности `Student`, `Teacher`, `Library` и др.
- **university.py**: Инкапсулирует бизнес-логику проведения экзаменов, добавления объектов, расчетов.
- **storage.py**: Работает с файлом `university.json` и преобразует объекты Python в JSON и обратно.
- **exceptions.py**: Классы ошибок (в веб-версии перехватываются базовым классом `UniversityError` для вывода понятных сообщений в браузере).

## 4. Ключевые алгоритмы и реализация

### Архитектурный паттерн
Проект реализован с оглядкой на паттерн **MVC (Model-View-Controller)**:
- **Model**: Пакет `sources` (данные и правила бизнес-логики).
- **View**: Файлы в `Web/templates/` (отвечают только за отображение информации).
- **Controller**: Файл `Web/app.py` (обрабатывает ввод от пользователя и связывает View с Model).

### Использование уникальных идентификаторов (ID)
При работе с веб-интерфейсом исключена проблема дублирования имен. В выпадающих списках (`<select>`) для удаления студентов или выдачи книг пользователю отображается ФИО (`full_name`), однако в качестве значения (`value`) на сервер отправляется скрытый уникальный `student_id`. Сервер производит поиск и удаление объектов строго по их `id`, сгенерированному через `uuid4()`.

### Обработка исключений в Web
Вместо аварийного завершения или принтов в консоль, веб-сервер перехватывает все исключения `UniversityError` (например, "Нет преподавателя по предмету" или "Студент не найден") в блоках `try-except`. Текст ошибки передается в функцию `flash()` с категорией `"danger"`, после чего пользователь перенаправляется (`redirect`) обратно на страницу, где видит красивое красное всплывающее уведомление.

## 5. Инструкция по запуску
### Требования
Python 3.8+ и установленный пакет Flask.

### Установка зависимостей
Если Flask не установлен, выполните команду в терминале:
```bash
pip install Flask
```

### Запуск приложения
Находясь в корневой директории проекта, выполните:
```bash
python Web/app.py
```
После запуска в терминале появится сообщение вида:
`* Running on http://127.0.0.1:5000`

Откройте любой веб-браузер и перейдите по указанному адресу `http://127.0.0.1:5000` для начала работы с системой. Все изменения будут сохранены в общем файле `university.json`.

*(Опционально)* Для проверки работы общего кода можно запустить консольную версию из соседней папки:
```bash
python CLI/main.py
```
Оба интерфейса будут корректно взаимодействовать с одной и той же базой данных.
