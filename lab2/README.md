# Лабораторная работа №2: Приложение для учёта пропусков студентов

**Документация GUI-приложения для учёта пропусков**

## 1. Общее описание
Данная программная система представляет собой GUI приложение для учёта пропусков занятий студентами. Система позволяет добавлять, удалять, искать и просматривать записи о студентах и количестве пропущенных ими занятий по различным причинам.

Проект реализован на языке Python с использованием библиотеки PyQt6 для построения графического интерфейса. Архитектура приложения следует паттерну Model-View-Controller (MVC), что обеспечивает разделение бизнес-логики, данных и их представления.

## 2. Структура проекта
Файловая структура проекта организована следующим образом:

```
lab2/
├── controller/             # Контроллер (связующее звено)
│   └── Controller.py       # Класс Controller
├── model/                  # Модель (данные и бизнес-логика)
│   └── Model.py            # Класс Model
├── view/                   # Представление (графический интерфейс)
│   ├── DeleteDialog.py     # Диалог удаления записей
│   ├── InputDialog.py      # Диалог добавления новой записи
│   ├── MainWindow.py       # Главное окно приложения
│   ├── PaginationWidget.py # Виджет для постраничной навигации
│   └── SearchDialog.py     # Диалог поиска записей
├── main.py                 # Точка входа в программу
└── .gitignore              # Исключения для Git
```

## 3. Описание модулей
### 3.1. main.py
Точка входа в приложение. Этот скрипт инициализирует `QApplication`, создает экземпляры `Model`, `MainWindow` (View) и `Controller`, связывает их вместе и запускает главный цикл обработки событий приложения.

### 3.2. Пакет model
#### Model.py
Отвечает за управление данными и всю бизнес-логику.
- Хранит основной список записей о студентах.
- Реализует методы для выполнения CRUD-операций: добавление (`add_record`), удаление (`delete_records`), поиск (`search_records`).
- Содержит логику для сортировки (`sort_records`) и постраничного получения данных (`get_paged_records`).
- Реализует сохранение и загрузку данных в форматах SQLite (`.db`) и XML.
- Использует механизм сигналов PyQt (`data_changed`) для уведомления других частей приложения об изменениях в данных, обеспечивая их своевременное обновление.

### 3.3. Пакет view
Содержит все классы, отвечающие за графический интерфейс.

#### MainWindow.py
Основной класс представления.
- Создает и настраивает главное окно приложения, меню, панель инструментов.
- Включает в себя `QTabWidget` с двумя вкладками: табличное представление (`QTableWidget`) и иерархическое (`QTreeWidget`).
- Содержит виджет постраничной навигации `PaginationWidget`.
- Определяет набор сигналов (`add_record_signal`, `delete_records_signal` и др.) для информирования контроллера о действиях пользователя.
- Имеет методы (`update_table_view`, `update_tree_view`) для обновления отображаемых данных при получении сигнала от модели.

#### PaginationWidget.py
Инкапсулирует логику и элементы управления для постраничной навигации.
- Позволяет переключаться между страницами, переходить к первой/последней странице и настраивать количество записей на странице.
- Генерирует сигнал `page_changed`, который сообщает приложению, какой срез данных необходимо отобразить.

#### InputDialog.py
Модальное окно для добавления новой записи.
- Предоставляет поля для ввода ФИО, группы и количества пропусков.
- Осуществляет базовую валидацию введенных данных (например, запрет пустых полей для ФИО и группы).

#### SearchDialog.py
Модальное окно для поиска записей по сложным критериям.
- Имеет три вкладки для разных типов поиска: по группе/фамилии, по типу пропуска, по диапазону количества пропусков.
- Выполняет поиск через модель и отображает результаты в собственной таблице с пагинацией.

#### DeleteDialog.py
Модальное окно для удаления записей по заданным критериям. По структуре и логике получения критериев похоже на `SearchDialog`.

### 3.4. Пакет controller
#### Controller.py
Класс-посредник, который связывает Модель и Представление.
- Подписывается на сигналы от `MainWindow` (например, `add_record_signal`) и вызывает соответствующие методы `Model` (`model.add_record`).
- Подписывается на сигнал `model.data_changed` и вызывает методы `MainWindow` для обновления интерфейса.
- Обрабатывает операции, требующие взаимодействия с пользователем через диалоговые окна, например, выводит сообщения об успехе или ошибке (`QMessageBox`).

## 4. Ключевые алгоритмы и реализация
### Паттерн Model-View-Controller (MVC)
Приложение строго следует архитектуре MVC:
- **Model**: Независима от интерфейса, содержит только данные и логику их обработки.
- **View**: Отвечает только за отображение данных и передачу действий пользователя в виде сигналов.
- **Controller**: "Склеивает" модель и представление, обеспечивая их взаимодействие.

### Механизм сигналов и слотов
Взаимодействие между компонентами построено на системе сигналов и слотов Qt. Это позволяет создавать слабую связь между объектами: например, `MainWindow` не знает о существовании `Model`, она лишь испускает сигнал, который `Controller` связывает с нужным слотом модели.

### Постраничная навигация (Pagination)
Для эффективной работы с большими объемами данных реализована пагинация. `PaginationWidget` запрашивает у модели не все записи сразу, а только ту часть (`offset`, `limit`), которая нужна для отображения на текущей странице. Это снижает нагрузку и улучшает отзывчивость интерфейса.

### Сохранение и загрузка
Реализована возможность сохранения всего набора данных в файл и последующей загрузки. Поддерживается два формата:
1.  **SQLite**: бинарный, быстрый и эффективный формат базы данных.
2.  **XML**: текстовый, легко читаемый человеком формат, удобный для обмена данными.

## 5. Инструкция по запуску
### Требования
- Python 3.x
- PyQt6

Для установки зависимости выполните команду:
```bash
pip install PyQt6
```

### Запуск приложения
Находясь в корневой директории проекта, выполните команду:

```bash
python main.py
```
